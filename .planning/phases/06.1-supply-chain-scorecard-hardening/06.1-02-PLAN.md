<task type="auto">
  <name>Task 2: Smoke-test gitleaks locally with the new config</name>
  <files></files>
  <action>
    Run gitleaks locally against the working tree with the new config to confirm zero NEW findings vs the pre-existing `.gitleaksignore`-baseline. The CI gate (the existing `secret-scan` job in ci.yml) will run the same scan in CI on PR — the local smoke test catches divergence before commit.

    Steps:
    1. Confirm gitleaks is installed: `command -v gitleaks` (if missing, `brew install gitleaks` on macOS).
    2. Run: `gitleaks detect --config .gitleaks.toml --no-banner --source .`
    3. Expected exit code: 0 (no findings) OR 1 with ONLY findings already present in `.gitleaksignore` baseline.
    4. If new findings surface that are NOT in the `.gitleaksignore` baseline, STOP — surface them to Yves. Do NOT add blanket suppressions or new `.gitleaksignore` entries from this plan (Phase 6 owns `.gitleaksignore`).
  </action>
  <verify>
    <automated>
      set -e
      if ! command -v gitleaks >/dev/null 2>&1; then
        echo "SKIP: gitleaks not installed locally; CI will run the same scan on PR"
        exit 0
      fi
      # exit 0 → zero findings; exit 1 → findings (acceptable only if all match .gitleaksignore baseline)
      gitleaks detect --config .gitleaks.toml --no-banner --source . > /tmp/gitleaks.out 2>&1
      RC=$?
      cat /tmp/gitleaks.out
      if [ "$RC" -eq 0 ]; then
        echo "OK: gitleaks zero findings with new config"
        exit 0
      fi
      # If RC != 0, check that all reported findings are already in .gitleaksignore.
      # Gitleaks prints the fingerprint of each finding; cross-check against .gitleaksignore.
      NEW=$(grep -Eo '[0-9a-f]{40}:[^[:space:]]+:[0-9]+' /tmp/gitleaks.out | sort -u || true)
      if [ -z "$NEW" ]; then
        echo "WARN: gitleaks exited $RC but no fingerprint pattern matched output — review manually"
        exit 1
      fi
      UNKNOWN=$(comm -23 <(echo "$NEW") <(sort -u .gitleaksignore))
      if [ -n "$UNKNOWN" ]; then
        echo "FAIL: gitleaks reports NEW findings not in .gitleaksignore:"
        echo "$UNKNOWN"
        exit 1
      fi
      echo "OK: all gitleaks findings already in .gitleaksignore baseline"
    </automated>
  </verify>
  <done>
    Local gitleaks scan with new config returns either zero findings OR only findings that match the existing `.gitleaksignore` baseline. No new suppressions added.
  </done>

- **Pitfall 4 — Wrong key name:** the inheritance key is `[extend]` `useDefault`, not `[extend.useDefault]` or `[defaults]`. Verify by reading gitleaks' README before committing.
