<objective>
Implement `src/http.lua` per D-25, Risk R-1, and Pitfall 1: a single-Connection module-local transport with `post_form` and `get_json` that always set `Accept: application/json` (mandatory shield against HTTP-4xx/5xx aborts), redact request and response bodies through `M_log.redact` before any DEBUG log, and infer HTTP status from the response body shape (the only signal available since `Connection():request` returns no status). `shutdown` closes the connection on `EndSession`.

Purpose: Without `Accept: application/json`, any HTTP 4xx/5xx from `oauth.zettle.com/token` aborts the entire Lua chunk and AUTH-03 (synchronous LoginFailed) fails. Without body-shape status inference, `M_errors.from_http_status` (Plan 02-02) cannot route to the LoginFailed / network / rate_limit branches. Without `shutdown`, `EndSession` cannot release the Connection per D-25. This plan is the load-bearing single seam where MoneyMoney's `Connection()` semantics meet our code.

Output: A ~70-LoC `src/http.lua` (`post_form`, `get_json`, `shutdown`, `_infer_status`, private `_get_connection` / `_form_encode` / `_merge_headers`); a small Wave-2 instrumentation addition (`Mocks._last_request`) in `spec/helpers/mm_mocks.lua` so specs can introspect HTTP call shapes; and a fully greened `spec/http_spec.lua` covering all ten pending Wave-0 tests.
</objective>

    - Builds form body via `_form_encode(body_table)` with sorted keys, MM.urlencode per key+value, joined by `&`
    - Calls `_get_connection():request("POST", url, body, "application/x-www-form-urlencoded", _merge_headers(headers))`
    - Destructures the FIVE returns; ignores the middle three (charset, mime, filename) but names them
    - Emits one DEBUG log on request: `M_log.debug("POST " .. url .. " body=" .. M_log.redact(body))` AND one DEBUG log on response: `M_log.debug("POST " .. url .. " response=" .. M_log.redact(raw))`
    - On empty content returns `(nil, nil, "")` (network anomaly per D-24)
    - JSON-parses content inside pcall; on parse failure returns `(nil, nil, raw)`
    - On success returns `(parsed, M_http._infer_status(parsed), raw)`

    - Same shape but `"GET"`, nil body, nil contentType
    - DEBUG request log is JUST `"GET " .. url` — NEVER any reference to headers (defense-in-depth against Bearer leakage)
    - DEBUG response log: `"GET " .. url .. " response=" .. M_log.redact(raw)`

    - Returns a NEW table; copies user_headers; sets `["Accept"] = "application/json"` unconditionally

    `M_http._form_encode(t)`:
    - Sorted keys, `MM.urlencode(k) .. "=" .. MM.urlencode(t[k])`, joined by `&`

    - Documented Risk R-1 contract above

    - `parsed.error == "invalid_client"` or `"unauthorized_client"` → 401
    - Other `parsed.error` non-nil → 400 (conservative; Pitfall 5)
    - Otherwise → 200

    `M_http.shutdown()`:
    - Closes `_conn` if non-nil AND has `close` method; sets `_conn = nil`; idempotent
